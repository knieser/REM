Introduction

This vignette introduces the use of the robust expectation maximization (REM) algorithm; a statistical method for estimating the parameters of a latent variable model in the presence of population heterogeneity as suggested by Nieser & Cochran (2021).The REM algorithm is based on the expectation-maximization (EM) algorithm, but it allows for the case when not all the data are generated by the assumed data generating model.

Data Description

To show how the REM package could be used for Exploratory and Confirmatory Factor Analysis with the Robust Expectation Maximization algorithm, here we utilize the Holzinger and Swineford data set available in the lavaan package. This data set is composed of 9 variables which correspond to mental ability test scores of seventh and eight-grade children. These variables are visual perception, cubes, Lozenges, paragraph comprehension, sentence completion, word meaning, speeded addition, speeded counting of dots and speeded discrimination straight and curved capitals. Additionally, the dataset contains other variables like an identifier (id), Gender (sex), age in years (ageyr), age in months (agemo), school (school) and grade (grade). The original dataset, which contains scores for 26 tests is available in the MBESS package (add the reference).

Preprocessing

The REM package requires data in a matrix form and to select variables associated with the scores needed in the factor analysis. In the case of the selected data set, columns 7 to 5 are the ones needed in the analysis as shown below:

data=df[,7:15]

Modeling using REM for Exploratory Factor Analysis

The REM_EFA function, which is used for exploratory factor analysis requires data in a matrix form, to specify the k_range and the delta. The k_range is a vector with the range of number of factors that you want to model. In model below, the algorithm will perform exploratory data analysis for 2, 3 and 4 factors. Additionally, the delta parameter is a hyperparameter between 0 and 1 that captures the researcher’s tolerance of incorrectly down-weighting data from the model. The output of the model will contain three different lists, each one for the number of factors being considered respectively.

model_EFA = REM_EFA( X = data, k_range = 2:4, delta = 0.05)

Compare EM and REM Estimates

library(dplyr)
library(ggplot2)

 x <- model_EFA
  k <- vector()
  j = length(x) - 1
  BIC <- vector(length = j)

  for (i in 1:j) {
    k[i] = x[[i]]$k
    BIC[i] = x[[i]]$BIC_rem
  }

  idx = which.min(BIC)
  optimal = x[[idx]]$k
  epsilon = x[[idx]]$epsilon
  gamma = x[[idx]]$REM_output$gamma
  EM <- cbind(x[[idx]]$EM_output$nu,x[[idx]]$EM_output$lambda,x[[idx]]$EM_output$psi)
  REM <- cbind(x[[idx]]$REM_output$nu,x[[idx]]$REM_output$lambda,x[[idx]]$REM_output$psi)

  colnames(REM) <- c("Intercept",paste("Factor",1:k[idx]),"Res Var")
  colnames(EM) <- c("Intercept",paste("Factor",1:k[idx]),"Res Var")


  ans <- x["call"]
  ans$optimal <-  optimal
  ans$EM <- EM
  ans$REM <- REM
  ans$epsilon <- epsilon
  ans$gamma <- gamma
  class(ans) <- "summary.REM"
  
diff = as.data.frame(ans$EM-ans$REM)

Confirmatory Factor Analysis

From the previous section we determine that the optimal number of factors should be three. For this reason, we will model a confirmatory factor analysis using the function REM_CFA using three factors (k = 3). In this case we need to specify X (numeric dataframe considered), k (number of factors for the confirmatory factor analysis), delta (hyper-parameter) and constraints (p x k matrix of constraints).
